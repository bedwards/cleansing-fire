Development Workflow

All work flows through this cycle. No exceptions.

The Definitive Development Loop

1
GitHub Issue â€” every piece of work starts as an issue with labels
2
Implementation Worker â€” Claude Opus 4.6, git worktree, feature branch cf/<issue>-<desc>, creates PR
3
Review Worker â€” separate Opus 4.6 instance, own worktree. Reviews PR. NEVER fixes code.
4
Fix Worker â€” if review finds issues, a separate worker reads comments and fixes. Iterate until green.
5
Orchestrator merges â€” resolves conflicts, verifies main branch and deployments are green.
6
Close the issue.
# Key rules
Workers:     Always Claude Opus 4.6, always git worktrees
Review:      NEVER fixes code â€” only reviews
Fix:         Separate from implementation and review
Merge:       Only the orchestrator merges
Verify:      GitHub Pages + Cloudflare Workers must be green
